Lección Técnica: La Naturaleza Estática de los Arrays

¿Por qué los Arrays en Java no pueden cambiar de tamaño?

Cuando creas un Array, Java reserva un bloque de memoria contiguo e inmutable. Explora estas 3 simulaciones físicas para entender cómo funciona la memoria RAM.

1. Inicialización Fija en Memoria Contigua

Al instanciar un array de tamaño 4, el Sistema Operativo asigna inmediatamente 4 casillas consecutivas. Las casillas vecinas quedan bloqueadas por otros procesos del sistema.

// Se reservan 4 espacios contiguos en la RAM int[] edades = new int[4]; edades[0] = 18; edades[1] = 25; edades[2] = 32; edades[3] = 40;

2. El Error de Desbordamiento (Fuera de Límites)

¿Qué pasa si intentamos añadir un 5to elemento en la posición edades[4]? Como el tamaño es estático, la casilla contigua pertenece a otra aplicación y Java bloquea el acceso por seguridad.

3. La Solución Algorítmica: Redimensión Manual (Resize)

Para simular un crecimiento dinámico, los desarrolladores debemos crear un array completamente NUEVO en otra dirección de memoria, clonar los datos viejos e insertar el nuevo elemento.

int[] viejo = {18, 25, 32, 40}; // Creamos una nueva estructura con tamaño 5 int[] nuevo = new int[5]; for(int i=0; i<viejo.length; i++) { nuevo[i] = viejo[i]; // Clonando... } nuevo[4] = 55; // Insertando el nuevo
