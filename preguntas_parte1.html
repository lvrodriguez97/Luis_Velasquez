Reto Avanzado 11: Inversión de Memoria (In-Place)

Analiza el siguiente método diseñado para invertir el orden de un array sin crear un segundo array auxiliar (optimización de espacio $O(1)$):

public static void invertir(int[] arr) { for (int i = 0; i < arr.length / 2; i++) { int temp = arr[i]; arr[i] = arr[arr.length - 1 - i]; arr[arr.length - 1 - i] = temp; } }

Si le pasamos el array `{5, 9, 3, 7, 2}`, ¿qué pasaría si cambiáramos la condición del ciclo a solo i < arr.length?

El array se invertiría dos veces más rápido debido al doble de iteraciones.
El array terminaría en su estado original: {5, 9, 3, 7, 2}.
El código lanzará un ArrayIndexOutOfBoundsException a la mitad del recorrido.
El array se llenaría con copias del elemento central: {3, 3, 3, 3, 3}.
[PISTA] Simula el comportamiento del código paso a paso para i = 0 e i = 4 en un papel y observa qué pasa con las posiciones físicas del extremo izquierdo y derecho.
Reto Avanzado 12: Técnica de Dos Punteros (Two-Pointers)

Tenemos un algoritmo optimizado que busca si existen dos números en un array ordenado que sumen un valor `target` específico:

int[] nums = {2, 5, 8, 12, 19}; int target = 20; int izq = 0, der = nums.length - 1; while (izq < der) { int suma = nums[izq] + nums[der]; if (suma == target) break; if (suma < target) izq++; else der--; }

¿Cuáles serán los valores finales de las variables de control `izq` y `der` tras finalizar la ejecución de este bloque?

izq = 0, der = 4
izq = 2, der = 3
izq = 3, der = 2
izq = 1, der = 2
[PISTA] Al estar ordenado, si la suma actual es mayor que el target, la única forma de reducir el resultado es mover el puntero derecho hacia abajo (der--).
Reto Avanzado 13: Desplazamiento Cíclico (Rotación)

Se requiere rotar los elementos de un array un casillero hacia la **derecha**, haciendo que el último elemento pase a ser el primero. Analiza este intento de implementación:

int[] items = {10, 20, 30, 40}; int ultimo = items[items.length - 1]; for (int i = 0; i < items.length - 1; i++) { items[i + 1] = items[i]; } items[0] = ultimo;

¿Qué error de lógica severo presenta este código?

El último elemento original se pierde porque se sobreescribe antes de guardarlo.
Provoca un ArrayIndexOutOfBoundsException en la última iteración del bucle.
El primer elemento "propaga" su valor destruyendo los datos siguientes; el resultado final será {40, 10, 10, 10}.
Produce un bucle infinito porque la longitud dinámica del array se altera.
[PISTA] Si ejecutas el código en orden, cuando i = 0, items[1] pasa a valer 10. Cuando i = 1, evalúas items[2] = items[1]. ¿Cuánto valía items[1] en ese instante? Para solucionar esto, el recorrido debería hacerse de atrás hacia adelante.
Reto Avanzado 14: Algoritmos de Frecuencias Histograma

Disponemos de un array con valores numéricos entre 0 y 4. Queremos contar cuántas veces se repite cada número usando un **array de frecuencias**. Observa la sintaxis:

int[] datos = {2, 4, 2, 0, 2, 4}; int[] frec = new int[5]; for (int i = 0; i < datos.length; i++) { frec[datos[i]]++; }

¿Qué técnica fundamental de arrays demuestra la línea resaltada en azul?

Uso del valor de un elemento como índice directo de acceso para otro array.
La creación de una referencia multidimensional en tiempo de ejecución.
Un incremento ilegal de punteros contiguos de memoria de solo lectura.
Una sintaxis alternativa exclusiva de Java para clonar arrays.
[PISTA] En la primera vuelta i = 0, datos[0] vale 2. Por lo tanto, la línea se traduce en ejecutar internamente: frec[2]++;.
Reto Avanzado 15: Traza de Ordenamiento (Bubble Sort)

Se ejecuta la **primera pasada completa** (el ciclo más externo completo, desde $j=0$ hasta terminar su primera ronda) del algoritmo de ordenamiento de burbuja clásico sobre este array desordenado:

int[] n = {5, 1, 4, 2, 8};

¿Cómo quedan posicionados los elementos del array tras completarse únicamente esa primera iteración global?

{1, 4, 2, 5, 8}
{1, 5, 4, 2, 8}
{1, 4, 5, 2, 8}
[PISTA] Recuerda el principio de la burbuja: compara parejas consecutivas (n[i] > n[i+1]) de izquierda a derecha. El elemento más pesado siempre es empujado hasta el final del array en la primera ronda.
